1. ¿Qué es Python?

Python es un lenguaje de programación versátil y fácil de aprender, creado en 1989 por Guido van Rossum. Se destaca por su sintaxis clara y legible, lo que lo hace perfecto para principiantes.

1. ¿Qué es una variable?

Imagina que una variable es como una "caja" o una etiqueta en la memoria de la computadora donde puedes guardar información para usarla más tarde.

nombre = "Fer"
edad = 20
# Ahora la caja 'nombre' guarda el texto "Fer"

En Python, no necesitas decirle de qué tipo es la caja, simplemente le asignas un valor usando el signo de igual (=).

2. Reglas para nombrarlas

No puedes nombrar tus cajas como quieras. Python tiene reglas estrictas:

  • No usar espacios: Usa guion bajo (ej. mi_edad).
  • No empezar con números: 1_nombre causará error.
  • Distinción: Edad y edad son variables diferentes.

Convención: En Python se acostumbra usar letras minúsculas separadas por guiones bajos (snake_case) para las variables.

3. Reasignación y uso con print()

El valor de una variable puede cambiar (por eso se llaman "variables"). Además, puedes usar print() para ver qué hay dentro de la caja.

puntos = 10
print(puntos) # Muestra 10

puntos = 50 # Cambiamos el valor
print(puntos) # Ahora muestra 50

1. Operadores Matemáticos

Además de la suma (+) y resta (-), Python tiene operadores especiales:

  • Exponente (**): Eleva un número a una potencia (ej: 2 ** 3 es 8).
  • Módulo (%): Devuelve el residuo de una división (ej: 10 % 3 es 1).

Se usan para comparar dos valores y siempre devuelven un Booleano (True o False):

3. Operadores Lógicos

Permiten combinar comparaciones:

  • and: True si AMBOS son verdaderos.
  • or: True si al menos UNO es verdadero.
  • not: Invierte el valor (True se vuelve False).

1. El bloque "if" (Si...)

La declaración if le dice a Python que ejecute un bloque de código solamente si una condición es verdadera (True).

edad = 18
if edad >= 18:
    print("Eres mayor de edad")

2. El bloque "else" (Si no...)

El else captura cualquier caso donde la condición del if fue falsa. No lleva ninguna condición al lado.

clima = "lluvia"

    print("Salgamos a pasear")
else:
    print("Mejor nos quedamos en casa")

3. La regla de Oro: Indentación

En otros lenguajes se usan llaves {}, pero Python usa espacios (indentación) para saber qué código está dentro del if. Si olvidas los espacios, Python te dará error.

Recuerda: Siempre que escribas dos puntos :, la siguiente línea debe estar indentada (usualmente con 4 espacios o la tecla Tab).

Un ciclo for sirve para repetir una acción varias veces o para recorrer una colección de datos letra por letra, o elemento por elemento.

La variable que pones después del for (como la i) toma el valor de cada paso del ciclo y puedes usarla dentro del bloque.

ORO 2: Ciclo While

Progreso del Curso

1. Bucle por Condición

A diferencia del for que sabe cuándo parar, el while (mientras) se repite infinitamente mientras una condición sea verdadera.

vidas = 3
while vidas > 0:
    print("Sigo jugando")
    vidas = vidas - 1

2. El peligro: Bucles Infinitos

Si olvidas actualizar la variable que controla la condición (como restar vidas en el ejemplo anterior), el bucle nunca terminará y tu programa se quedará trabado (crasheará).

3. Break y Continue

Puedes alterar el flujo: break rompe el ciclo por completo y sale de él. continue salta el resto del código en esa vuelta y regresa al inicio del ciclo.

Puedes asignarle un valor predeterminado a un parámetro. Si quien llama a la función no envía ese dato, se usará el valor por defecto.

La forma más segura de abrir un archivo en Python es usar with open(). Esto garantiza que el archivo se cierre automáticamente cuando termines.

MAESTRO 2: Herencia

Progreso del Curso

0%

1. Clases Hijas (Heredar)

La herencia permite crear una nueva clase que absorbe todos los atributos y métodos de una clase existente (Clase Padre). Se usa para no repetir código.

class Animal:
    def comer(self): print("Comiendo...")

class Gato(Animal): # Gato hereda de Animal
    def maullar(self): print("Miau")

michi = Gato()
michi.comer() # Puede usar el método de su Padre

2. Sobreescritura (Override)

Si la Clase Hija tiene un método con el mismo nombre que la Clase Padre, la Hija "sobrescribe" el método. El suyo tendrá prioridad.

3. La función super()

A veces quieres sobrescribir el __init__ del padre para añadir algo nuevo, pero no quieres perder lo que el padre ya hacía. super() te permite invocar métodos del Padre desde la Hija.

super().__init__(nombre) ejecutará el constructor del Padre para que la Hija no tenga que reescribir esa lógica.

3. Decoradores (@)

Un decorador es una función que "envuelve" a otra función para darle nuevas capacidades antes o después de ejecutarse sin modificar el código original. Se usan con el símbolo @.

Por ejemplo, en Flask (el framework de BushiCode) usamos el decorador @app.route('/') para conectar una función con una URL de la web.

1. Generadores (yield)

Cuando trabajas con millones de datos, una Lista consume toda tu memoria RAM. Un Generador crea los datos uno por uno sobre la marcha usando la palabra yield.

def contador_infinito():
    n = 0
    while True:
        yield n
        n += 1
